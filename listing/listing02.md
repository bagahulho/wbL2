Что выведет программа? Объяснить вывод программы. Объяснить как работают defer’ы и их порядок вызовов.

```go
package main

import (
	"fmt"
)


func test() (x int) {
	defer func() {
		x++
	}()
	x = 1
	return
}


func anotherTest() int {
	var x int
	defer func() {
		x++
	}()
	x = 1
	return x
}


func main() {
	fmt.Println(test())
	fmt.Println(anotherTest())
}
```

Ответ:
```
Будет выведено 2 и 1.

В первом случае будет 2, x - именнованое возвращаемое значение,
и так как дефер может читать и изменять именнованное возвращаемое значение, он увеличит значение на единицу.

Во втором случае будет 1, x - неименнованное возвращаемое значение, после чего return сохранит значение x,
но так как дефер не может влиять на неименнованное возвращаемое значение, он не сможет увеличить возвращаемое значение на 1.

Когда объявляется defer, то его функция помещается в стек отложенного вызова, который будет выолнене после выхода непосредственно из тела функции.
Из этого следует:
1. Порядок выполнения деферов LIFO
2. Возможность изменять именнованное возвращаемое значение, так как возврат самого значения происходит после дефера
3. Сама функция defer вычисляется после об его объявления
```
